# Дерево / структура

## Оргструктура (оптимизация без расширения штата)

```mermaid
flowchart TB
  OWN[Собственники] --> COO[Опердир (COO)]
  COO --> SALES[РОП / продажи]
  COO --> PROD[Продуктолог]
  COO --> PROC[Закупщик]
  COO --> WMS[ОМ МойСклад / складской учёт]
  COO --> FIN[Финансист]
  COO --> AST[Ассистент]

  SALES <--> PROD
  PROC <--> WMS
  FIN <--> COO
  SALES <--> FIN
```

## Принципы (чтобы работало без найма)
1. **Опердир держит контур прибыли:** план‑факт, реклама, цены, остатки, риски.
2. **РОП отвечает за продажи и выполнение планов:** реклама/контент/карточки — через KPI и дедлайны.
3. **Продуктолог = ассортимент + карточки + SEO:** правки через трекер, «не трогаем без ТЗ».
4. **Закупщик = поставка и сроки:** ETA, риски, закрытие «узких мест».
5. **ОМ МойСклад = точность учёта:** приходы/перемещения/инвентаризация, чтобы финансы и продажи не «летали».
6. **Финансист = ДДС + P&L + контроль расходов:** прозрачность кассы и маржи.
7. **Ассистент = скорость и чистота выполнения:** протоколы, напоминания, документооборот, сбор данных.

> Если мы видим «пожары» — это **сломанный контур**: нет дедлайнов, нет отчёта, нет владельца процесса.
